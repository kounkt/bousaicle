# 02. 企画書

## プロダクト名

**ボウサイクル**(仮称・第1候補)
- 由来: 防災 × サイクル(ローリングストック=備える→食べる→買い足す のサイクル)
- サブコピー: 「チエロと3分、わが家の備え。」

候補一覧(公開前に商標・既存サービス重複チェック必須 → [08_release_plan.md](08_release_plan.md)):
1. ボウサイクル(Bousaicle)— 推奨。コンセプトを名前が説明する
2. そなえチエロ — キャラクター一体型。検索独自性が高い
3. わが家のそなえ — 説明的で検索に強いが汎用的すぎる

以下、本ドキュメント群では「ボウサイクル」で統一。

## ワンライナー

> 住まいと家族構成を入れるだけ。3分で「わが家に必要な備え」がリストになり、賞味期限までチエロが面倒を見てくれる無料Webアプリ。

## 解決する課題(Problem)

「地震が不安。でも何から手を付ければいいかわからない。一度買った備蓄も、気づいたら賞味期限切れ。」

- 不安はある(臨時情報認知80%・不安70%超)のに行動できない(新規備蓄26.7%)
- 非備蓄理由1位「何から手を付ければいいかわからない」(約5割)
- 備えた人も平均3.32日分で止まり(推奨1週間)、ローリングストック実施は4人に1人
- 既存ツールは「診断で終わる」(東京備蓄ナビ)か「管理が重すぎる」(備蓄管理アプリ)

## 解決のかたち(Solution)— 難しいことの抽象化

防災の専門知識(ハザード、備蓄量基準、ローリングストック法、避難計画)を、ユーザーには **3つの動詞** だけに抽象化して見せる:

1. **しらべる**(3分診断)— 質問に答えるだけで「わが家専用の備えリスト」と「そなえスコア」が出る
2. **そろえる**(リスト消化)— 優先度順チェックリスト。「今週末の買い物のついでに水2L×6本」のように、行動を日時・場所つきの具体文に変換(if-thenプラン)
3. **まわす**(ローリングストック)— 賞味期限を登録すると、切れる前に「そろそろ食べて、買い足そう」と知らせる。冷蔵庫に貼る1枚シートも生成(環境省RCTの実証手法)

ユーザーが学ぶべき専門用語はゼロ。エビデンスは裏側に埋め込む。

## ターゲットとペルソナ

**プライマリ: 30〜40代・子のいる世帯の家庭運営担当者**
> 佐藤美咲(36)・横浜市・共働き・子2人(5歳/8歳)。臨時情報のとき水を買い足したきり。「ちゃんとしなきゃ」と思いつつ、何が足りないのか分からないまま1年経過。家計簿アプリは挫折経験あり。判断はスマホ、買い物は週末にまとめて。
- ニーズ: 正解を教えてほしい/続けられる仕組み/家族(夫)と分担したい
- 刺さる価値: 3分で「わが家の正解」が出る。買い物リストとして使える。シートを冷蔵庫に貼って家族と共有

**セカンダリ: 20〜30代・一人暮らし**
> 田中蓮(26)・大阪市・賃貸ワンルーム。「何から手を付ければいいかわからない」層のボリュームゾーン(20代の約半数)。SNSで診断系コンテンツに反応する。
- 刺さる価値: そなえスコアの画像シェア。「最低限これだけ」の小さく始められるリスト

**サード: 50〜60代・親世代/実家が心配な子世代**
- 刺さる価値: 印刷シート(デジタル苦手な親に渡せる)。「親に送る」導線

## 競合との差別化

| | ボウサイクル | 東京備蓄ナビ | 備蓄管理アプリ各種 | Yahoo!防災/NERV |
|---|---|---|---|---|
| 診断(必要量算出) | ◎ 全国対応 | ◎(東京都発・全国利用可) | ✗ | ✗ |
| 継続管理・期限通知 | ◎ 診断結果がそのまま管理リストに | ✗ | ○(手入力が重い) | ✗ |
| 登録不要・Web完結 | ◎ | ◎ | ✗(アプリDL+登録) | ✗ |
| 行動への翻訳(if-then) | ◎ 買い物計画・避難プラン | ✗ | ✗ | ✗ |
| 家庭内共有(印刷シート) | ◎ 環境省RCT準拠 | ✗ | △ | ✗ |
| 速報・通知 | ✗(やらない。完成形が既にある) | ✗ | ✗ | ◎ |

ポジショニング: **「速報のNERV、備えのボウサイクル」**。速報系とは補完関係であり、むしろ相互送客できる(リスト内からNERV等のインストールを推奨する)。

## ビジネスモデル

- **完全無料・広告なしで公開**(チエロの作品として。信頼がブランド資産)
- サーバーコストほぼゼロの設計(静的ホスティング+localStorage)なので無料を維持できる
- 将来オプション(MVP では実装しない): 防災グッズのアフィリエイトリンク(その場合は広告表記を明示)。寄付ボタン

## KPI

| 段階 | 指標 | 目標(公開3ヶ月) |
|---|---|---|
| 獲得 | ユニーク訪問者 | 10,000 |
| 活性 | 診断完了率(開始→結果表示) | 70%以上 |
| 価値実感 | リスト保存(localStorage書き込み)率 | 診断完了者の50% |
| 拡散 | スコア画像のシェア/結果URLコピー率 | 診断完了者の10% |
| 継続 | 30日後の再訪率 | 15% |
| 行動変容(本丸) | チェックリスト消化率・シートDL数 | 計測して基準値を作る |

## 拡散設計(マーケティング)

1. **そなえスコア画像**: 診断結果を「わが家のそなえスコア 42点」のシェア画像に。ボタンメーカー型の自己診断系拡散(X・Instagramストーリーズ・LINE)
2. **イベントドリブン拡散**: 地震・台風のたびに防災ツイートが急増する。災害時に「今すぐ確認」ではなく「落ち着いたら3分で」と添えて再共有される構造(押し付けないトーンが重要)
3. **9月1日(防災の日)・3月11日・11月5日(津波防災の日)** に合わせたコンテンツ更新
4. SEO: 「備蓄 必要量 計算」「ローリングストック やり方」「防災 何から」等のハウツー検索流入。診断ツール+解説記事のセット
5. チエロのキャラクター性(ゆるい・ちょい生意気・知的)がそのまま発信トーンになる: 「ふふん、まかせてよ。」

## リスクと対策

| リスク | 対策 |
|---|---|
| 数量基準の誤りで信頼失墜 | 公的基準(東京備蓄ナビ・農水省ストックガイド)に準拠し出典明記。「目安」であることを常時表示 |
| 災害時にアクセス集中 | 静的サイト+CDN(Cloudflare)なので落ちにくい。災害時モード(軽量表示)も検討 |
| 「防災を商売にしている」批判 | 完全無料・広告なし。アフィリエイト導入時は明示 |
| 名称の商標・重複 | 公開前チェックを必須タスク化 |
| 不安を煽る表現 | トーンガイドで禁止(→ [06_brand.md](06_brand.md))。恐怖訴求は行動変容効果も低いことが研究で示されている |
