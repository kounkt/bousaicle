# 04. 画面設計・UXフロー・マイクロコピー

## 画面一覧

| ID | 画面 | 役割 |
|---|---|---|
| S0 | ランディング(トップ) | 価値を10秒で伝え、診断開始ボタンへ。再訪者にはダッシュボード表示 |
| S1 | 診断(5問) | 1問1画面、進捗バー、戻れる |
| S2 | 結果(スコア+リスト) | そなえスコア、わが家のリスト、シェア、保存 |
| S3 | リスト管理 | チェックリスト(持ってる/持ってない/今週そろえる)、if-thenプラン作成 |
| S4 | まわす(期限管理) | 期限登録、食べごろ通知、ICS出力 |
| S5 | 冷蔵庫シート | 集合場所等の入力 → 印刷/PNG |
| S6 | 設定・データ | エクスポート/インポート、共有リンク、全削除、出典、プライバシー |

## メインフロー

```
S0 → [そなえチェックをはじめる] → S1(Q1..Q5) → S2(スコア+リスト)
                                                ├─ [画像でシェア]
                                                ├─ [リストを使う] → S3 ⇄ S4
                                                └─ [冷蔵庫に貼るシートを作る] → S5
再訪: S0がダッシュボード化(スコア、期限アラート、今週のプラン)
```

設計原則:
- **初回価値まで最短**: S0→S2 を90秒以内。登録・メール・許可ダイアログを一切挟まない(摩擦削減ナッジ d≈0.29 の適用)
- **1画面1判断**: 診断は1問ずつ。リストは3状態トグルのみ
- **デフォルト効果**: 推奨値(7日分)を初期選択。質問の選択肢も最頻値を先頭に
- **lapse-recovery**: 期限切れ・放置を責めない。再訪自体を歓迎する
- **煽らない**: 恐怖訴求・地震動画・被害写真は使わない。「不安→安心」ではなく「わからない→わかる」の感情設計

## チエロのマイクロコピー集(トーン: ゆるい×知的×ちょい生意気)

| 場面 | コピー |
|---|---|
| 診断開始 | 「ふふん、まかせてよ。3分で終わらせるから」 |
| 診断中の励まし | 「いい感じ。あと2問」 |
| スコア低(0-20) | 「おっ、まっさらだね。つまり、これから全部良くなるってこと」 |
| スコア中(40-60) | 「半分まで来てる。水とトイレだけ先に埋めよ。そこが本体」 |
| スコア高(80+) | 「…やるじゃん。あとは『まわす』だけ。それができたら本物」 |
| if-then保存時 | 「『いつ・どこで』まで決めた人は、実行率が段ちがいなんだよね」 |
| 期限切れ | 「切れちゃったか。だいじょうぶ、気づけたのが偉い。買い物リストに戻しといた」 |
| 30日ぶり再訪 | 「おかえり。サボっても戻ってくる人がいちばん強いんだよ」 |
| 災害ニュース直後(検討) | 「不安なときに頑張らなくていい。落ち着いたら、3分だけやろ」 |

禁止表現: 「あなたの家は危険です」「〜しないと後悔します」等の恐怖・脅し訴求。断定的な安全保証(「これで安心」)も不可 → 「目安」「公的情報も確認してね」を添える。

## 状態設計(空・エラー・ロード)

- 空状態(S3で全部「持ってない」): チエロ「ゼロからが一番伸びる。まず水だけ、今週そろえよ」
- localStorage不可: 上部に常設バナー「保存できないモードで動いてるよ。リストはスクショしてね」
- 共有リンク受信時: 「〇〇さんの備えリストを受け取ったよ。自分用にコピーする?」(上書き確認必須)
- オフライン(v1.1): キャッシュ済みリストを表示+「オフラインでも見られるよ」

## アクセシビリティ実装メモ

- 診断の選択肢は `<button>`(role=radio のカスタムではなく実ボタン)で十字キー/タブ移動可能に
- スコアは色だけでなく数値+文言で伝える(色覚多様性)
- チエロの台詞は装飾でなくコンテンツとして読み上げ対象に(aria-hidden にしない)
- 印刷シートは白黒印刷でも成立するデザイン(赤→濃グレーに落ちても読める)
