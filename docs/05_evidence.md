# 05. エビデンス集 — 研究→機能マッピング

本アプリの機能は「効果が実証された介入手法」を一般の人が意識せず使える形に翻訳したもの。本書がその対応表と、サイト内でエビデンスを語るときの原則。

## 一般向け翻訳の原則(サイト内でエビデンスを語るとき)

1. **数字は正直に、専門用語は出さない**: 「実行率が倍ちがう」のように体感語に訳す。「d=0.65」「メタ分析」は本文に出さない(出典ページには書く)
2. **断定しない**: 「〜になりやすいことが研究でわかってる」止まり。「必ず」「絶対」は禁止
3. **過大宣伝しない**: ナッジ研究には出版バイアスの指摘があり、効果量は控えめに見るのが誠実。「科学の力で完璧に備わる」とは言わない
4. **出典は1クリック以内**: フッターに「この数字の根拠」ページを常設。研究リンクと公的ガイドを一覧
5. **チエロが言う**: 知識はキャラクターの口から。「『いつ・どこで』まで決めた人は実行率が段ちがいなんだよね」=実装意図研究の翻訳
6. **限界も言う**: 「このリストは目安。お住まいの自治体の情報も確認してね」を結果画面に常設
