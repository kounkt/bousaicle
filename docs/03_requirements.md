# 03. 要件定義

## スコープ方針

「機能を増やさない勇気」(個人開発ヒット事例の共通項)。MVP は **しらべる/そろえる/まわす** の3動詞を最短で成立させる7機能のみ。速報・通知・SNS機能・アカウントは永久にやらない(non-goals)。

## 機能要件

### MVP(F-01〜F-07)

**F-01 3分診断(しらべる)**
- 入力(全5問、選択式のみ・テキスト入力なし):
  1. お住まいの都道府県(+任意で市区町村)→ 地域リスク文脈(ハザードマップポータルへのリンク生成)
  2. 住まいのタイプ: 戸建て / マンション低層(1-2F) / マンション中高層(3F以上)→ 在宅避難前提度・水の運搬・トイレ重要度に影響
  3. 大人の人数(うち65歳以上の人数)
  4. 子どもの人数(乳幼児 0-2歳 / 3歳以上 を区別)
  5. ペット(犬・猫・その他・なし)、食物アレルギーや常用薬の有無(チェックのみ)
- 出力: 必要備蓄リスト(F-02)+そなえスコア(F-03)
- 所要時間目標: 60秒以内で回答完了(「3分」はリスト確認込み)

**F-02 わが家の備蓄リスト生成**
- 計算ロジック(出典: 東京備蓄ナビの考え方・農水省「災害時に備えた食品ストックガイド」に準拠。実装前に最新版で数値再確認):

| 品目 | 計算式(1週間分推奨・最低3日分も併記) | 備考 |
|---|---|---|
| 飲料水 | 3L × 人数 × 日数 | 乳幼児はミルク用軟水を別途 |
| 主食(米・パックご飯・麺) | 3食 × 人数 × 日数 | 普段食べるものでOK(ローリングストック前提) |
| 主菜(缶詰・レトルト) | 1〜2品 × 人数 × 日数 | |
| 簡易トイレ | 5回 × 人数 × 日数 | 集合住宅は特に優先度高 |
| カセットコンロ+ボンベ | 1台+ボンベ約6本/週(4人世帯目安) | |
| 乳幼児用品 | おむつ・ミルク・離乳食 × 日数 | 乳幼児ありの場合のみ |
| 高齢者用品 | 常用薬リスト・お薬手帳コピー、介護食 | 65歳以上ありの場合のみ |
| ペット用品 | フード・水 × 日数、トイレ用品 | ペットありの場合のみ |
| 共通防災用品 | モバイルバッテリー、ライト、ラジオ、救急セット、現金(小銭)、ウェットティッシュ、ラップ等 | 固定リスト |
- 各品目に「優先度(S/A/B)」を付与。S=命に直結(水・トイレ・薬)、A=1週間の生活、B=快適性
- 日数は 3日(最低)/ 7日(推奨)をトグル切替。デフォルト7日(デフォルト効果 d=0.68 の応用 — 推奨値を初期選択にする)

**F-03 そなえスコア**
- リストの所持チェック状況から 0〜100点を算出(優先度Sの重みを大きく)
- 初回診断時は「いま何点か」を5問の所持ざっくり質問で推定 → チェックを進めると上がる
- スコア帯ごとにチエロの表情・コメントが変わる(5段階。煽らない文言 → [04_ux_screens.md](04_ux_screens.md))
- シェア画像生成(Canvas): スコア+チエロ+「あなたも3分でチェック」+URL。**家族構成等の個人情報は画像に含めない**

**F-04 チェックリスト管理(そろえる)**
- リスト項目ごとに: 持ってる / 持ってない / 今週そろえる の3状態
- 「今週そろえる」に入れた項目は **if-thenプラン文** に変換するUIを出す:
  - テンプレート: 「(いつ)____ に(どこで)____ のついでに ____ を買う」
  - 例: 「土曜の午前、OKストアでの買い出しのついでに 水2L×6本 を買う」
  - 選択式で文章が完成する(自由入力は任意)。完成文は読み上げ表示+保存(実装意図のリハーサル効果)
- データはすべて localStorage。登録・ログインなし

**F-05 ローリングストック管理(まわす)**
- 賞味期限の登録: 品目ごとに「年+月」だけ(日まで入れさせない=摩擦削減)。バーコードスキャンはMVPでは非対応
- 期限30日前になると: サイト再訪時にトップで「そろそろ食べごろ」通知 + バッジ表示
- **ICSカレンダー出力**: 「期限1ヶ月前に通知」イベントをまとめて .ics でダウンロード/Googleカレンダー追加リンク(プッシュ通知サーバーを持たないための設計。→ 07_tech.md)
- 期限切れが出ても責めない: 「切れちゃったね。食べられるかは見た目と匂いで判断…はダメ。処分して、買い物リストに戻しておいたよ」(lapse-recovery設計: 復帰を称える)

**F-06 冷蔵庫シート生成(印刷用1枚もの)**
- 環境省RCT(日本・1,200世帯で防災行動が有意に改善)の「常時目に入る掲示物」をWebで再現
- A4縦1枚に: 家族の集合場所 / 災害用伝言ダイヤル171の使い方 / わが家の備蓄チェック表(手書きチェック欄付き) / 避難先メモ / チエロ
- 集合場所・避難先は診断後に選択式+自由入力で設定
- 出力: 印刷(print CSS)+ PNG保存。**シートに入れた個人情報はサーバーに送信されない**(完全クライアントサイド生成)

**F-07 データの持ち運び**
- エクスポート/インポート: JSON ファイル
- 家族共有: 状態を圧縮してURLハッシュに載せる共有リンク(サーバー不要)。「夫のスマホにもこのリストを入れる」ユースケース

### v1.1 以降(MVP外・優先順)
1. PWA化(ホーム画面追加・オフライン閲覧)— 災害時はオフラインで備蓄リストが見られるべき
2. Web Push通知(要小規模バックエンド or 外部サービス。費用と相談)
3. 多言語(やさしい日本語 → 英語)
4. 季節リスク(台風前チェックリスト等)
5. 速報系アプリ(NERV等)への送客コーナー「備えたら、次は速報アプリを」

### Non-goals(永久にやらない)
- 地震速報・気象通知(NERV/Yahoo!が完成形)
- アカウント登録・サーバーへの個人データ保存
- SNS・コミュニティ機能
- 有料プラン

## データモデル(TypeScript)

```ts
type Household = {
  version: 1;                    // スキーマバージョン(マイグレーション用)
  prefecture: string;
  dwelling: 'house' | 'apt_low' | 'apt_high';
  adults: number;                // 65歳未満
  seniors: number;               // 65歳以上
  kidsInfant: number;            // 0-2歳
  kidsChild: number;             // 3歳以上
  pets: ('dog' | 'cat' | 'other')[];
  flags: { allergy: boolean; medication: boolean };
  targetDays: 3 | 7;
};

type StockItem = {
  id: string;                    // 品目マスタID or カスタム
  name: string;
  category: 'water' | 'staple' | 'main' | 'toilet' | 'heat' | 'baby' | 'senior' | 'pet' | 'common';
  priority: 'S' | 'A' | 'B';
  requiredQty: number;           // 計算結果
  unit: string;
  status: 'have' | 'need' | 'this_week';
  expiry?: { year: number; month: number };  // 月まで
  plan?: string;                 // if-then 文
};

type AppState = {
  household: Household | null;
  items: StockItem[];
  sheet: { meetingPoint?: string; evacSite?: string; memo?: string };
  meta: { createdAt: string; updatedAt: string; lastVisit: string };
};
// localStorage キー: "bousaicle:v1"
```

## 非機能要件

| 項目 | 要件 |
|---|---|
| プライバシー | 個人データはブラウザ内(localStorage)のみ。サーバー送信ゼロ。解析はクッキーレス(→ 07_tech.md)。この事実を画面に明示(信頼の核) |
| パフォーマンス | 初回ロード 150KB(gzip)以下目標、Lighthouse Performance 95+。災害時の弱い回線でも開ける |
| アクセシビリティ | WCAG 2.1 AA。キーボード操作完結、コントラスト比4.5:1以上(赤#E60012を背景に使う場合は白文字の比率確認)、スクリーンリーダー対応。NERVの先例=防災×a11yは信頼に直結 |
| 対応環境 | iOS Safari / Android Chrome 直近2年+PC各ブラウザ。スマホファースト(375px基準) |
| オフライン | MVP: 通常キャッシュ。v1.1: PWAで完全オフライン化 |
| 可用性 | 静的サイト+CDN。災害時のスパイクに耐える(サーバー処理ゼロが最大の対策) |
| 正確性 | 数量基準はすべて出典つき。年1回の定期見直しタスク(→ 08_release_plan.md 運用) |

## エッジケース

- 人数0人で診断 → 大人1人にフォールバック+メッセージ
- localStorage 不可(プライベートブラウズ等)→ その場で使える・保存だけできない旨を非破壊的に通知
- localStorage 容量・破損 → try/catch + スキーマバリデーション(zod)で安全に初期化、破損データはエクスポート退避を提案
- 共有URLのデータが巨大 → lz-string 圧縮 + 上限超過時はJSONファイル共有へ誘導
- 期限が過去の月で登録された → 確認ダイアログ(入力ミス対策)
